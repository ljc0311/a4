# 🚀 一键发布功能文档

## 📖 功能概述

一键发布功能允许用户将生成的视频一次性发布到多个短视频平台，包括B站、抖音、快手、小红书、微信视频号和YouTube Shorts。该功能采用模块化设计，支持视频格式自动转换、内容智能优化、多账号管理和发布状态监控。

## 🏗️ 系统架构

### 核心组件

1. **平台发布器 (Platform Publishers)**
   - `BasePublisher`: 抽象基类，定义通用接口
   - `BilibiliPublisher`: B站发布器实现
   - `PublisherFactory`: 发布器工厂类

2. **视频格式转换器 (Video Format Converter)**
   - 支持多平台格式适配
   - 基于FFmpeg的高质量转换
   - 批量处理能力

3. **数据库服务 (Database Service)**
   - 任务管理
   - 账号管理
   - 发布记录
   - 统计分析

4. **一键发布服务 (One Click Publisher)**
   - 统一发布流程
   - 并发处理
   - 错误处理和重试

5. **用户界面 (GUI)**
   - 发布配置界面
   - 账号管理界面
   - 状态监控界面

## 🚀 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 启动程序

```bash
python main.py
```

### 3. 配置账号

1. 点击导航栏中的 "🚀 一键发布"
2. 切换到 "账号管理" 标签页
3. 点击 "添加账号" 按钮
4. 选择平台并输入认证信息

### 4. 发布视频

1. 在发布配置界面选择视频文件
2. 填写视频信息（标题、描述、标签）
3. 选择目标平台
4. 点击 "开始发布"

## ⚙️ 配置说明

### 数据库配置

```python
DATABASE_CONFIG = {
    'url': 'sqlite:///data/publisher/publisher.db',
    'echo': False,
    'pool_size': 5,
    'max_overflow': 10
}
```

### 平台配置

```python
PLATFORM_CONFIG = {
    'bilibili': {
        'enabled': True,
        'api_base': 'https://api.bilibili.com',
        'timeout': 300,
        'max_retries': 3
    }
}
```

### 视频转换配置

```python
VIDEO_CONVERSION_CONFIG = {
    'ffmpeg_path': 'ffmpeg',
    'temp_dir': 'temp/video_conversion',
    'max_concurrent_conversions': 3,
    'quality_preset': 'medium'
}
```

## 🔐 账号认证

### B站认证 (Cookie方式)

1. 在浏览器中登录B站
2. 按F12打开开发者工具
3. 切换到Network标签页
4. 刷新页面，找到主页面请求
5. 在Request Headers中找到Cookie字段
6. 复制Cookie值到账号配置对话框

### 其他平台认证

其他平台的认证方式类似，都是通过Cookie或API密钥进行认证。具体步骤请参考账号配置对话框中的说明。

## 🛠️ 开发指南

### 添加新平台支持

1. 创建新的发布器类，继承`BasePublisher`
2. 实现必要的抽象方法
3. 在`PublisherFactory`中注册新平台
4. 添加平台规格到`VideoFormatConverter`
5. 更新GUI界面的平台选择

### 扩展认证方式

1. 在`BasePublisher`中添加新的认证方法
2. 更新账号配置对话框
3. 实现相应的凭证存储和验证逻辑

### 自定义视频处理

1. 扩展`VideoFormatConverter`类
2. 添加新的平台规格
3. 实现自定义的FFmpeg命令生成逻辑

## 🐛 故障排除

### 常见问题

1. **FFmpeg未找到**
   - 确保FFmpeg已安装并在PATH中
   - 或在配置中指定FFmpeg的完整路径

2. **数据库连接失败**
   - 检查数据库文件权限
   - 确保SQLAlchemy等依赖已安装

3. **平台认证失败**
   - 检查Cookie是否过期
   - 确认账号状态正常
   - 验证网络连接

4. **视频转换失败**
   - 检查输入视频文件是否损坏
   - 确认有足够的磁盘空间
   - 查看详细错误日志

### 日志查看

程序运行时会生成详细的日志文件，位于：
- 主程序日志：`logs/app.log`
- 发布器日志：`logs/publisher.log`
